Bring Your Own
Disease — as FAIR Data

Capture everything you know about a rare disease: clinical phenotypes, onset timelines, patient organisations, expert panels, literature, social media communities — and make it permanently machine-readable.

A candid note before we start.  This is the first time this system runs with real contributors. We have built a complete FAIR Data Point backend, per-disease versioned datasets, and a full form-to-RDF pipeline — but this is the inaugural run. Expect some start-up friction. We also face a classic chicken-and-egg problem: there are no prior submissions to show, because you are the first.
How to participate

Sign in with your ORCID and fill in one of the web forms. Your submission creates a GitHub Issue instantly, and the system converts it to RDF Turtle and publishes it as a FAIR dataset within about 60 seconds.

Open the session page
Practical recommendation

Also keep a document or notes while you work — unstructured is fine. If the system has a hiccup we can always add your notes later. The goal is to not lose the momentum or the knowledge you bring today.

What is in this system

Five contribution forms · a FAIR Data Point · per-disease versioned datasets · a live SPARQL endpoint

Contribution forms

Five ways to contribute

  • Submit a disease case — narrative, timeline, ontology IDs, data types
  • Submit a disease resource — paper, patient org, expert panel, social media community
  • Report an ontology gap — missing or misaligned terms in ORDO, HPO, ICD, SNOMED
  • Report a data model gap — limitations in Phenopackets, FHIR, OMOP, registries
  • Feedback on the forms — propose changes or entirely new forms

FAIR Data Point

Machine-readable output

  • Every submission becomes an RDF Turtle file within 60 seconds
  • Each disease gets its own versioned DCAT Dataset
  • Catalog published as Turtle and JSON-LD — ready to load into any RDF tool
  • Per-disease pages with cases, ontology gaps, and data gap reports linked together
  • Registered in the StaticFDP index — discoverable worldwide

Live query

SPARQL in the browser

  • Powered by Comunica — fully client-side, no server needed
  • Query all registered FAIR Data Points from one interface
  • Pre-loaded queries: list diseases, find by ORDO or OMIM ID, list submissions
  • Works on any FDP that follows the DCAT / FDP spec
  • As you submit today, queries will start returning your data
